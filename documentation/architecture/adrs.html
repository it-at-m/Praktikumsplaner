# Architecture Decision Records (ADRs)

Architecture decision records present one method to document functional and non-functional decisions relating to architecture. We use the ADR template of Michael Nygard (opens new window).

The following ADRs exist:

  • ADR-001 use final
  • ADR-002 dont use @Column-annotation
  • ADR-003 test as prefix for test case methods
  • ADR-004 Reduce mapstructmapper tests to a minimum
  • ADR-005 dtos as java records
  • ADR-006 frontend components without complex logic
  • ADR-007 send errormessage
  • ADR-008 security roles
